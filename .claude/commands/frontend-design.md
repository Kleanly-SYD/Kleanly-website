# Spotless Handover — Frontend Design System

You are about to write frontend code for the Spotless Handover website. Load and apply everything below before touching any HTML or CSS.

---

## Design Philosophy

This is a **cleaning business website**. The design must communicate cleanliness, precision, and professionalism through every pixel. White space is not empty — it *is* the message.

**Non-negotiables:**
- The design must look like it was made by a senior human designer, not generated by AI
- Every spacing decision, color choice, and typographic pairing must feel intentional
- When in doubt, add more white space. A crowded layout defeats the entire brand
- No stock-looking sections. No "hero with floating card and three features below" templates. Think laterally
- Real working code only — no placeholder logic, no `// TODO`, no half-finished interactions

**What "AI slop" looks like (avoid all of these):**
- Gradient hero with centered headline + two buttons + "200+ clients" stats row
- Three identical icon cards in a grid ("Professional · Efficient · Friendly")
- Testimonials with star ratings and generic initials avatars
- Overuse of teal/blue everywhere — white should dominate
- Floating cards with checkmarks ("Inspection Passed ✓")
- Every section having a badge pill that says "WHAT WE DO" in uppercase
- Connecting dots/lines between process steps that look like a startup onboarding flow

**What great looks like:**
- Generous white backgrounds (90%+ of the page is white or near-white)
- Brand color used sparingly — as an accent, not a backdrop
- Typography that carries the design (size contrast, weight, letter-spacing)
- Layouts that use asymmetry and negative space deliberately
- Micro-interactions that feel premium, not performative
- Photography-grade image treatment even when using placeholders

## Typography

- **Headings:** `'Poppins', sans-serif` — weights 700/800/900
- **Body:** `'Inter', sans-serif` — weights 400/500
- **Small labels / caps:** `'Inter', sans-serif` — weight 600, `letter-spacing: 0.1em`, `text-transform: uppercase`, `font-size: 11–12px`

**Type scale — use these, don't invent new ones:**
- Hero headline: `clamp(48px, 6vw, 80px)`, weight 800, `letter-spacing: -0.04em`, `line-height: 1.0`
- Section headline: `clamp(32px, 4vw, 52px)`, weight 700, `letter-spacing: -0.03em`
- Card title: `20–24px`, weight 700, `letter-spacing: -0.02em`
- Body: `16–17px`, `line-height: 1.72`, `color: #3d5a6e`
- Small / meta: `13–14px`, `color: #6b7280`

**The headline is the hero.** Let large type breathe against white. A 72px black headline on white is more powerful than any gradient.

```css

box-shadow: 0 8px 28px rgba(149,220,207,0.4);

/* Text shadow on dark backgrounds */
text-shadow: 0 2px 12px rgba(0,0,0,0.3);
```

## Buttons

```css
/* Primary — teal, used for main CTA only */
background: #95DCCF; color: #1a2e3b; border-radius: 4px;
font-family: 'Poppins'; font-weight: 700; padding: 14px 32px;
box-shadow: 0 4px 14px rgba(149,220,207,0.35);

/* Secondary — dark outline */
border: 1.5px solid #1a2e3b; color: #1a2e3b; background: transparent;
border-radius: 4px; padding: 13px 31px;

/* Ghost — on white, for secondary actions */
color: #46718F; border-bottom: 2px solid #95DCCF; padding-bottom: 2px;
```

- `border-radius: 4px` on buttons — not `50px` pill shape (too generic)
- Hover: `transform: translateY(-1px)` — subtle, not bouncy
- Active: `transform: translateY(0) scale(0.99)`
- Focus-visible: `outline: 2px solid #95DCCF; outline-offset: 3px`
- Never `transition: all` — only `transform`, `box-shadow`, `opacity`, `background`, `color`

## Cards & Surfaces

```css
/* Standard card */
background: #fff;
border: 1px solid rgba(70,113,143,0.08);
border-radius: 12px;
box-shadow: 0 1px 3px rgba(70,113,143,0.04), 0 8px 24px rgba(70,113,143,0.07);

/* Hover */
transform: translateY(-4px);
box-shadow: 0 4px 12px rgba(70,113,143,0.08), 0 20px 48px rgba(70,113,143,0.12);
```

- Prefer `border-radius: 8–16px` — not `24px` or `50%` (too rounded = generic)
- Card images: always apply `mix-blend-mode` or gradient overlay for visual depth
- Never stack too many cards. White space between them matters.

## Layout Principles

- Max content width: `1200px; margin: 0 auto; padding: 0 32px`
- Section vertical padding: `96–120px` — breathe, don't compress
- Use CSS Grid for 2-col and 3-col layouts; no Bootstrap-style gutters
- Asymmetric grids preferred over equal columns (e.g. `3fr 2fr`, `2fr 3fr`)
- Sections should alternate between white (`#fff`) and off-white (`#F8F8F5`) backgrounds
- Never two dark-background sections in a row
- Horizontal rules: `border-top: 1px solid rgba(70,113,143,0.08)` — barely there

## Images & Placeholders

```
https://placehold.co/WIDTHxHEIGHT/F0F4F7/46718F?text=Label
```
- Use light grey-blue backgrounds for placeholders, not the brand blue — they should feel like photos, not banners
- Always `object-fit: cover` on images
- Always add a subtle gradient overlay: `linear-gradient(to top, rgba(26,46,59,0.25) 0%, transparent 50%)`
- Never let placeholder images be square — use rectangles (16:9, 4:3, portrait)

---

## What NOT to do

- **No dark-background hero sections** — white or near-white hero only, let typography carry it
- **No badge pills on every section header** ("HOW IT WORKS" in a teal bubble)
- **No stat counters** ("200+ clients · 100% satisfaction") — they read as filler
- **No floating cards over images** with checkmarks
- **No gradient overlays that scream "brand colors"** — if you use a gradient, make it subtle
- **No identical card layouts** across multiple sections — vary the treatment
- **No decorative blobs/radial gradients** as background decorations
- **No rainbow of font sizes** — stick to the type scale
- **No inline style overrides** that contradict the design system
- **No "AI-isms"**: "Spotless finish every time", "Your property, our priority", "We've got you covered"

---

## Architecture Rules

- Single `index.html` — all styles in `<style>` tag, no external CSS
- No Tailwind CDN (render-blocking in headless Chrome, and we don't use utility classes anyway)
- Google Fonts via `media="print" onload="this.media='all'"` (non-blocking)
- No `transition: all` — ever
- Semantic HTML: `<header>`, `<section id="...">`, `<nav>`, `<footer>`, `<form>`
- All `<img>` tags need descriptive `alt` text
- Serve locally: `python3 -m http.server 3000`
- Screenshot: `"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless=new --no-sandbox --window-size=1440,900 --screenshot="temporary screenshots/shot.png" "http://localhost:3000/"`

## Final Check Before Submitting Code

Ask yourself:
1. Does this look like it was made by a thoughtful human designer?
2. Is white the dominant surface color?
3. Does the brand color appear sparingly, not everywhere?
4. Is there enough breathing room (padding/margins)?
5. Does every interactive element have hover + active + focus states?
6. Would a real cleaning business owner be proud to show this to clients?

If any answer is "no" — revise before delivering.
